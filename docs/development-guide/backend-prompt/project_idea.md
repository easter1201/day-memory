# 🧠 프로젝트 아이디어

---

## 1. 프로젝트명

**Day Memory** - 기념일 관리 및 AI 기반 선물 추천 애플리케이션

---

## 2. 프로젝트 개요

Day Memory는 기념일과 이벤트를 효과적으로 관리하고, 사용자에게 적절한 시기에 리마인더를 제공하며, AI를 활용하여 관계와 기념일 유형에 맞는 맞춤형 선물을 추천하는 웹 애플리케이션입니다. 사용자가 저장한 선물 리스트를 우선적으로 표시하고, AI가 추가적인 선물 아이디어를 제공하여 선물 선택의 부담을 줄이고 소중한 기념일을 놓치지 않도록 돕습니다.

---

## 3. 개발 목적 및 배경

### 해결하고자 하는 문제
- **기념일 관리의 어려움**: 생일, 커플 기념일, 특별한 날 등 챙겨야 할 날들이 많아 놓치기 쉬움
- **선물 선택의 부담**: 기념일마다 어떤 선물을 준비해야 할지 고민되고 시간이 많이 소요됨
- **준비 미루기**: 미리 준비하려던 선물을 까먹거나 막판에 서두르게 되는 상황 발생

### 기술적 학습 목표
- **이벤트 스케줄링**: Spring의 `@Scheduled`를 활용한 정기적인 배치 작업 구현
- **AI API 통합**: OpenAI/Claude API를 활용한 지능형 추천 시스템 구축
- **성능 최적화**: N+1 문제 해결, 쿼리 최적화, 인덱싱 전략 학습
- **이메일 시스템**: JavaMailSender를 활용한 알림 발송 시스템 구현
- **Full Stack 개발**: Spring Boot + React를 이용한 전체 애플리케이션 개발 경험

---

## 4. 주요 기능 목록

### 4.1 사용자 인증 및 관리
- **회원가입 / 로그인**: JWT 기반 토큰 인증, Spring Security 활용
- **소셜 로그인**: Google, Kakao OAuth 연동 (선택)
- **프로필 관리**: 사용자 정보 수정

### 4.2 이벤트 / 기념일 관리
- **이벤트 CRUD**: 기념일 생성, 조회, 수정, 삭제
- **다양한 이벤트 타입 지원**:
  - 생일, 100일/200일/300일 기념일, 1주년 등
  - 월별 커플 데이 (발렌타인데이, 로즈데이, 화이트데이 등 14일 시리즈)
  - 특별한 날 (크리스마스, 새해, 빼빼로데이 등)
  - 휴가 및 사용자 정의 이벤트
- **D-Day 계산**: 실시간 남은 일수 표시
- **반복 이벤트**: 매년 반복되는 기념일 설정
- **이벤트 추적 설정**: 챙기고 싶은 기념일만 선택적으로 추적

### 4.3 유연한 리마인더 시스템
- **맞춤형 리마인더 설정**: 사용자가 원하는 일수 설정 (예: 30일, 14일, 7일, 3일, 1일 전)
- **자동 리마인더 발송**: 매일 오전 9시 스케줄러를 통한 자동 알림
- **이메일 알림**: JavaMailSender를 통한 이메일 발송
- **중복 발송 방지**: 24시간 이내 발송 이력 확인
- **리마인더 활성화/비활성화**: 이벤트별 리마인더 on/off 토글

### 4.4 선물 관리
- **선물 아이디어 저장**: 평소 생각해둔 선물 아이템 저장
- **카테고리별 분류**: 꽃, 주얼리, 화장품, 패션, 전자기기, 음식, 체험, 책, 취미용품 등
- **구매 여부 추적**: 선물 구매 완료 체크
- **이벤트 연결**: 특정 기념일과 선물 연결
- **가격 및 URL 저장**: 선물 정보 및 구매 링크 저장

### 4.5 AI 기반 선물 추천 (예정)
- **맞춤형 추천**: OpenAI/Claude API를 활용한 지능형 선물 추천
- **컨텍스트 기반**: 기념일 유형, 관계, 예산 등을 고려한 추천
- **우선 순위**: 사용자 저장 선물 리스트 우선 표시 → AI 추천 추가 제공
- **추천 이유 제공**: AI가 해당 선물을 추천한 이유 설명

### 4.6 대시보드 및 통계
- **다가오는 이벤트 표시**: 향후 30일 이내 이벤트 리스트
- **월별 이벤트 캘린더**: 캘린더 뷰로 한눈에 확인
- **통계 정보**: 이벤트 수, 선물 구매 내역 등 (예정)

## 5. 기대 효과 및 목표

### 사용자 관점
- **기념일 누락 방지**: 자동 리마인더로 중요한 날을 놓치지 않음
- **시간 절약**: AI 추천으로 선물 고민 시간 단축
- **스트레스 감소**: 체계적인 기념일 관리로 준비 부담 완화
- **선물 아이디어 축적**: 평소 떠오른 선물 아이디어를 바로 저장하여 관리

### 개발자 관점 (학습 목표)
- **스케줄링 및 배치 처리**: Spring의 `@Scheduled`를 활용한 정기 작업 구현 경험
- **AI API 통합**: OpenAI/Claude API 연동 및 프롬프트 엔지니어링 역량 향상
- **성능 최적화**:
  - N+1 쿼리 문제 해결 (fetch join 활용)
  - 데이터베이스 인덱싱 전략
  - 쿼리 최적화 기법
- **실무 중심 설계**:
  - REST API 설계 및 문서화
  - DTO 패턴 활용
  - 계층화 아키텍처 구현
- **이메일 시스템**: JavaMailSender를 활용한 알림 시스템 구현
- **상태 관리**: Redux Toolkit을 활용한 프론트엔드 상태 관리
- **Full Stack 개발**: 백엔드부터 프론트엔드까지 전체 애플리케이션 개발 경험
- **배포 경험**: Docker, CI/CD를 통한 실제 서비스 배포 경험

## 6. 예상 기술 스택

### Backend
- **언어**: Java 17+
- **프레임워크**: Spring Boot 3.x
- **ORM**: Spring Data JPA (Hibernate)
- **데이터베이스**: PostgreSQL
- **인증/인가**: Spring Security, JWT
- **이메일**: JavaMailSender (SMTP)
- **AI 통합**: OpenAI API 또는 Claude API
- **스케줄링**: Spring `@Scheduled`
- **빌드 도구**: Maven

### Frontend
- **언어**: TypeScript
- **프레임워크**: React 18+
- **상태 관리**: Redux Toolkit
- **스타일링**: CSS Modules / Styled Components / Tailwind CSS
- **HTTP 클라이언트**: Axios
- **빌드 도구**: Create React App / Vite

### DevOps & Tools
- **버전 관리**: Git, GitHub
- **컨테이너**: Docker, Docker Compose
- **CI/CD**: GitHub Actions
- **클라우드**: AWS (EC2, RDS, S3) 또는 GCP
- **모니터링**: Spring Boot Actuator (예정)

### 개발 도구
- **IDE**: IntelliJ IDEA (Backend), VS Code (Frontend)
- **API 테스트**: Postman
- **데이터베이스 도구**: DBeaver, pgAdmin

## 8. 핵심 비즈니스 로직

### 리마인더 발송 프로세스
1. 매일 오전 9시 스케줄러 실행
2. 향후 365일 이내 추적 중인 이벤트 조회
3. 각 이벤트의 활성화된 리마인더 확인
4. 오늘 발송해야 할 리마인더 필터링
5. 중복 발송 방지 (24시간 이내 발송 이력 확인)
6. 이메일 발송 및 로그 기록

### AI 선물 추천 프로세스 (예정)
1. 사용자 요청 수신 (이벤트 정보, 관계, 예산 등)
2. 사용자가 저장한 선물 리스트 우선 조회
3. AI API에 컨텍스트 전달 (이벤트 유형, 관계, 선호도 등)
4. AI 추천 결과 수신 및 파싱
5. 사용자 저장 선물 + AI 추천 통합하여 응답
